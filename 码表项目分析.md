# FreeRTOS 骑行码表旗舰版（4G）项目分析

## 一、系统整体架构

### 分层结构

```
[User Layer]    main.c, bike.c, screen.c, flash_data.c, flash_index.c
      ↓
[System Layer]  system.h (OS抽象: os_delay_ms, os_malloc, os_millis)
      ↓
[Driver Layer]  Drivers/ (OLED, AHT10, GPS, W25Qxx, AIR780E, key, led, usart)
      ↓
[RTOS Layer]    FreeRTOS kernel (tasks, queues, timers, event groups)
      ↓
[HAL Layer]     RTE/Device/STM32F103C8 (STM32 标准外设库)
```

### 硬件资源

| 资源 | 配置 |
|------|------|
| MCU | STM32F103C8 (Cortex-M3, 72MHz) |
| Flash | 64KB 片内 + 2MB W25Qxx 外部 |
| SRAM | 20KB |
| FreeRTOS Tick | 1000Hz (1ms) |
| 最大任务优先级 | 5 |
| FreeRTOS 堆 | 5120 字节 (heap_4) |

### 外设分配

| 接口 | 设备 | 驱动 |
|------|------|------|
| USART1 | 调试日志 | `System/log.c` |
| USART2 | AIR780E 4G 模块 | `Drivers/at_air780e.c` |
| USART3 | GPS 模块 | `Drivers/GPS.c` |
| I2C1 | AHT10 温湿度 + OLED 显示屏 | `Drivers/AHT10.c`, `Drivers/OLED.c` |
| SPI1 | W25Qxx SPI Flash | `Drivers/W25Qxx.c` |
| PA0 | LED 绿 | `Drivers/led.c` |
| PC13 | LED 红 | `Drivers/led.c` |
| PB0 | KEY1 (EXTI0) | `Drivers/key.c` |
| PC14 | KEY2 (EXTI15_10) | `Drivers/key.c` |

## 二、系统启动流程

```
上电复位
    ↓
NVIC_PriorityGroupConfig(NVIC_PriorityGroup_4)   // 4位抢占优先级
    ↓
led_init()      // PA0, PC13 GPIO 初始化
    ↓
log_init()      // USART1 115200 初始化
    ↓
bike_init()
  ├── key_init()         // EXTI 外中断
  ├── i2c_init()         // I2C1 总线
  ├── screen_init()      // OLED 初始化
  ├── bike_reset()       // 状态和数据初始化
  ├── init_flash()       // W25Qxx + 索引读取
  └── air780e_init()     // 4G 模块使能脚
    ↓
bike_start()
  ├── xTimerCreate(1000ms, timer_handle_bike_cb)  // 创建1秒定时器
  └── xTaskCreate(bike_task, 优先级3)             // 创建骑行主任务
    ↓
usart3_init()   // USART3 9600 初始化，启用 GPS 接收中断
    ↓
vTaskStartScheduler()   // 启动 FreeRTOS 调度器
```

## 三、骑行状态机

### 状态定义

```c
enum BIKE_STATE   { IDEL, RUNNING, STOP }
enum RUNNING_STATE{ RUNNING_NOT_CONNECTED_SERVER,
                    RUNNING_CONNECTED_SERVER,
                    RUNNING_UPDATED_TIME }
enum SCREEN_NO    { SCREEN_WELCOME, SCREEN_1, SCREEN_2 }
```

### 状态转换图

```
上电
  └→ IDEL（欢迎屏，等待 GPS 定位）
         ↓ KEY1 按下 + GPS 有效定位
      RUNNING（骑行中）
       ├── 每秒计算速度、距离、卡路里、温湿度
       ├── 每 10 秒写入一次 Flash
       ├── 每 2 秒上传数据到云端
       └── OLED 实时刷新
         ↓ KEY1 按下
       STOP（骑行结束）
       ├── 停止 4G 连接
       └── 保存最后一批数据到 Flash
         ↓ KEY1 按下
      IDEL（bike_reset 清零，返回欢迎屏）
```

### 状态转换函数

`bike_running_state_change()` (`User/bike.c`)：
- `IDEL → RUNNING`：需要 `gps_enable() == 1`，记录开始时间戳，启动 4G
- `RUNNING → STOP`：停止 4G，调用 `save_item(..., STOP)` 保存最后数据
- `STOP → IDEL`：调用 `bike_reset()` 清零所有字段

## 四、任务通知与中断处理

### 通知位定义 (`User/bike.h`)

```c
NOTIFY_KEY1_PRESS          (1 << 0)   // KEY1 短按/长按
NOTIFY_KEY2_PRESS          (1 << 1)   // KEY2 短按/长按
NOTIFY_GPRMC_RECV          (1 << 2)   // GPS GPRMC 帧到达
NOTIFY_CONNECTED_SERVER    (1 << 3)   // 4G 连接服务器成功
NOTIFY_TIME_UPDATE         (1 << 4)   // 时间同步完成
NOTIFY_DATA_LIST           (1 << 5)   // 数据列表请求
NOTIFY_DATA_EXPORT         (1 << 6)   // 数据导出请求
```

### 中断 → 通知 → 处理链

| 中断源 | 触发条件 | 通知位 | bike_task 处理 |
|--------|---------|--------|---------------|
| EXTI0 (KEY1) | PB0 下降沿 + 20ms消抖 | `NOTIFY_KEY1_PRESS` | `bike_running_state_change()` |
| EXTI15_10 (KEY2) | PC14 下降沿 + 20ms消抖 | `NOTIFY_KEY2_PRESS` | `bike_screen_change()` |
| USART3 (GPS) | 收到完整 `$GPRMC` 帧 | `NOTIFY_GPRMC_RECV` | `parse_GPRMC()` |
| USART2 (4G) | 收到时间响应 JSON | `NOTIFY_TIME_UPDATE` | 更新 `timestamp_local` |
| 软件 (at_air780e) | TCP 连接成功 | `NOTIFY_CONNECTED_SERVER` | 更新 `running_state` |

### bike_task 主循环

```c
while (1) {
    xTaskNotifyWait(0, 0xFFFFFFFF, &notify_bits, portMAX_DELAY);
    // 永久阻塞，收到任意通知后清除所有位，逐一处理
}
```

### 定时器回调（每秒执行）

```
timer_handle_bike_cb()
  ├── bike_screen_show()         // 刷新 OLED（所有状态）
  ├── bike_local_handle()        // 计算本地数据（仅 RUNNING）
  └── bike_online_handle()       // 云端交互（仅 RUNNING）
```

## 五、GPS 模块工作流程

### 数据结构

```c
struct {
    char  time_str[18];   // 北京时间 "24-06-27 14:47:55"
    float speed_kt;       // 速度（节）
    char  latitude[12];   // 纬度（度分格式 DDMM.MMMMMM）
    char  longitude[13];  // 经度（度分格式 DDDMM.MMMMMM）
    int   enable;         // 1=有效定位，0=无效
};
```

### 完整数据流

```
GPS 硬件（USART3, 9600bps）
    ↓ 每字节触发 USART3_IRQHandler
gps_data_revevied(rx_data)
    ↓ 收到 '\n' 且以 "$GPRMC" 开头
xTaskNotifyFromISR(NOTIFY_GPRMC_RECV)
    ↓ bike_task 唤醒
parse_GPRMC()
  ├── 字段 1: UTC 时间 → 转换为北京时间（+8h）
  ├── 字段 2: 定位状态 A/V
  ├── 字段 3-4: 纬度 + 方向
  ├── 字段 5-6: 经度 + 方向
  └── 字段 7: 速度（节）
    ↓
bike_calc_running_data() 中：speed_kmh = speed_kt × 1.852
```

### GPRMC 帧示例

```
$GPRMC,060322.00,A,4003.520920,N,11620.220964,E,0.354,,010624,,,A*72
        时间     状态  纬度      方向  经度       方向  速度    日期
```

## 六、AHT10 温湿度传感器工作流程

### 接口

```c
int aht10_read(float *humi, float *temp);  // 0=成功, -1=失败
```

### 读取流程（每秒轮询）

```
bike_local_handle() → aht10_read()
    ↓
I2C 写：发送测量命令 [0xAC, 0x33, 0x00]
    ↓
os_delay_ms(80)（等待测量完成，手册要求 ≥75ms）
    ↓
I2C 读：接收 6 字节原始数据
    ↓
校验 bit3（校准位）和 bit7（忙闲位）
    ↓
数据解算：
  湿度 = (SRH × 100) / 2^20     // SRH 为字节1-3的20位值
  温度 = (ST × 200) / 2^20 - 50 // ST 为字节3-5的20位值
    ↓
更新 my_bike.humi / my_bike.temp
    ↓
calc_average_humi_temp() 计算累积平均值
```

## 七、Flash 数据存储流程

### 2MB Flash 布局

```
0x000000 ─── INDEX_SECTOR0 (4KB)  ─── 索引区（双缓冲）
0x001000 ─── INDEX_SECTOR1 (4KB)  ─── 索引区（交替使用）
0x002000 ─── TABLE0  (31扇区, ~124KB)
0x021000 ─── TABLE1  (31扇区)
  ...
0x1E0000 ─── TABLE15 (32扇区, 128KB)
0x200000 ─── 结束（共 16 个 TABLE，最多保存 16 次骑行记录）
```

### 索引结构体

```c
struct flash_index {
    uint16_t ntables;           // 已保存的记录数（0-16）
    uint16_t next_table_i;      // 下一次写入的表下标（0-15）
    uint32_t table_ts[16];      // 每条记录的创建时间戳
    uint32_t next_version;      // 版本号（每写一条 +1，用于双缓冲判新旧）
    uint8_t  reserved[2];
    uint8_t  next_no;           // 0 或 1，指示下一次用哪个索引扇区
    uint8_t  write_flag;        // 0xA5 = 已写入
};
```

### 单条 Record（256 字节 = 1 Flash 页）

```
┌──────────────────────────────────┐
│ 13 × riding_item (234字节)       │  每 10 秒采集一条
│   double lon_deg  (8B)           │
│   double lat_deg  (8B)           │
│   uint16 speed_kmh (2B)          │
├──────────────────────────────────┤
│ 2 字节保留                        │
├──────────────────────────────────┤
│ riding_status (20字节)            │
│   uint32 timestamp               │
│   uint16 max_speed_kmh           │
│   uint16 average_speed_kmh       │
│   float  kcal                    │
│   uint16 average_humi            │
│   int16  average_temp            │
│   uint16 total_distance_km       │
│   uint8  reserved                │
│   uint8  write_flag (0xA5)       │
└──────────────────────────────────┘
```

### 写入流程

```
骑行进行中，每 10 秒调用 save_item(lat, lon, speed, state)
    ↓
缓存 riding_item 到 items[] 数组（最多 13 个）
    ↓
满 13 个 或 state == STOP
    ↓
get_riding_status()     // 从 my_bike 读取统计数据
    ↓
write_record_to_flash()
  ├── 组装 256 字节 record
  ├── 地址是 4KB 边界时擦除扇区（w25qxx_sector_erase）
  └── 写入页（w25qxx_page_write_bytes）
    ↓
首次写入时 write_index()
  ├── 擦除 INDEX_SECTOR0 或 INDEX_SECTOR1（交替）
  ├── 写入更新后的 flash_index 结构体
  └── 版本号 +1，next_no 取反
```

## 八、OLED 屏幕显示流程

### 三种屏幕布局

#### SCREEN_WELCOME（IDEL 状态）

```
行1: 日期（从 GPS 时间中提取）
行2:   Welcome to use
行3:    Guo Guo Bike
行4:   GPS NOT Ready!  （GPS 未定位时显示）
```

#### SCREEN_1（RUNNING / STOP 状态，KEY2 切换）

```
行1: 日期
行2: 实时速度    23.45 km/h
行3: 累计距离    12.34 km
行4: 骑行时间    00:45:30          go!
```

#### SCREEN_2（RUNNING / STOP 状态，KEY2 切换）

```
行1: 最大速度    45.67 km/h        go!
行2: 平均速度    30.12 km/h
行3: 卡路里      125.50 KCAL
行4: 温湿度      23.50C 45.30%
```

### 刷新流程

```
定时器每秒回调
    ↓
bike_screen_show()
    ↓
根据 my_bike.current_screen 选择显示函数
    ↓
screen_welcome_show() / screen_1_show() / screen_2_show()
    ↓
OLED_ShowString(行, 列, 字符串)  通过 I2C1 写入 OLED 显存
```

## 九、4G 模块（AIR780E）工作流程

### 连接建立序列

骑行开始后调用 `air780e_start(ip, port, callback)`，执行以下 AT 命令序列：

```
AT              检查模块就绪
ATE0            关闭回显
AT+CPIN?        检查 SIM 卡状态
AT+CREG?        检查运营商网络注册
AT+CGATT?       检查 GPRS 附着
AT+CSTT         设置 APN
AT+CIICR        激活 PDP 上下文
AT+CIPMODE=1    设置透传模式
AT+CIPSTART     TCP 连接服务器（IP: 8.135.10.183, Port: 34135）
    ↓
连接成功 → xTaskNotify(NOTIFY_CONNECTED_SERVER)
```

### 数据上传（每 2 秒，已同步时间后）

```
bike_cloud_upload_data()
    ↓
encode_bike_upload()
  └── cJSON 编码为 JSON 字符串
    ↓
air780e_send_data(json_str)  透传模式直发
```

上传 JSON 格式：

```json
{
  "cmd": "upload data",
  "name": "dev_bike_kieran",
  "type": "bike",
  "time": 1735376027,
  "data": {
    "temp": 24.05, "humidity": 22.64,
    "speed": 33.5, "distance": 10.44,
    "total_time": 120,
    "max_speed": 30, "average_speed": 30,
    "kcal": 30,
    "lat": 39.999861, "lng": 116.484648
  }
}
```

注：经纬度经过度分格式 → 十进制 → GCJ-02（火星坐标）转换。

### 时间同步

```
连接成功后（RUNNING_CONNECTED_SERVER）：每秒发送
  {"cmd": "time request"}
    ↓
服务器响应：{"cmd": "time response", "time": 1735376027}
    ↓
air780e_recv_data_cb() → bike_handle_server_data()
    ↓
timestamp_local = 云端时间戳 + 8×3600（转北京时间）
    ↓
xTaskNotify(NOTIFY_TIME_UPDATE)
    ↓
my_bike.running_state = RUNNING_UPDATED_TIME
  └── 改为每 2 秒上传数据，每 180 秒重新同步时间
```

### 断开连接

骑行停止时：

```
air780e_stop()
    ↓
发送 +++（退出透传，需前后各延迟 1000ms）
    ↓
AT+CIPCLOSE     断开 TCP 连接
AT+CIPSHUT      关闭 PDP 上下文
```

## 十、骑行数据计算

`bike_local_handle()` 每秒调用（仅 `RUNNING` 状态）：

| 数据项 | 计算方式 |
|--------|---------|
| 速度 | `gps_get_speed_kt() × 1.852` |
| 累计距离 | 每秒 `+= speed_kmh / 3600` |
| 累计时间 | 每秒 `total_time_sec++` |
| 最大速度 | `max(max_speed, speed_kmh)` |
| 平均速度 | `total_distance / total_time` |
| 卡路里 | 每秒 `+= speed_kmh × 0.2 / 1000` |
| 平均温湿度 | 累加均值，`reset_data` 标志控制重置 |

## 十一、关键时序参数

| 参数 | 值 | 说明 |
|------|----|------|
| FreeRTOS Tick | 1ms | 系统节拍 |
| 定时器周期 | 1000ms | 屏幕刷新、数据计算 |
| GPS GPRMC 间隔 | ~1s | GPS 每秒发送一帧 |
| AHT10 测量延迟 | 80ms | 发送命令后等待 |
| Flash 保存间隔 | 10s | 每 10 秒采集一条 riding_item |
| 每条记录容量 | 13 条 item | 满后写入一个 Flash 页（256B）|
| 数据上传间隔 | 2s | 时间同步后 |
| 时间同步间隔 | 1s（未同步）/ 180s（已同步） | |
| 按键消抖 | 20ms | 软件延迟消抖 |
| 长按判定 | 2000ms | KEY 持续按下超过 2s |

## 十二、关键数据结构

### bike 结构体（`User/bike.h`）

```c
struct bike {
    float    total_distance_km;     // 累计距离
    uint32_t total_time_sec;        // 累计时间（秒）
    float    speed_kmh;             // 实时速度
    float    max_speed_kmh;         // 最大速度
    float    average_speed_kmh;     // 平均速度
    float    humi;                  // 实时湿度
    float    temp;                  // 实时温度
    float    average_humi;          // 平均湿度
    float    average_temp;          // 平均温度
    float    kcal;                  // 消耗卡路里
    enum BIKE_STATE    bike_state;
    enum RUNNING_STATE running_state;
    enum SCREEN_NO     current_screen;
    uint8_t  reset_data;            // 统计重置标志
    uint8_t  data_export_n;         // 导出记录编号
};
```
